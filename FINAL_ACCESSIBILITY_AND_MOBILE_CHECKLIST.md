# Final Accessibility And Mobile Checklist

## Accessibility

- Headings follow a readable order within each module section.
- Inputs have visible labels.
- Buttons use native button elements and accessible names.
- Keyboard focus remains visible.
- Status, warning, and validation messages use text, not color alone.
- Current animation state is described with labels such as Current, Active path, Root, or Step.
- Reduced-motion users can step through animations without autoplay.
- Tables, grids, and byte/matrix visuals have nearby labels or captions where feasible.
- Guided and challenge controls are keyboard usable.

## Mobile

- Check at 360px width.
- No page-level horizontal overflow.
- Long hex, Base64, binary, PEM, and digest values wrap or scroll inside panels.
- Tables and matrices scroll inside their panels.
- Buttons remain touch-sized.
- Sticky mobile actions do not cover essential content.
- Safety warnings appear before risky inputs.
- Quiz, guided, challenge, and animation controls remain tappable.
